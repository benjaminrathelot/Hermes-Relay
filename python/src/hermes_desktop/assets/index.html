A local operator console for crisis messaging. This shell drives the Hermes Relay C core, keeps private material local, and helps an operator decide whether to route traffic over LAN, Internet relay, or physical file handoff when infrastructure degrades.

The UI is loopback-only. Plaintext exists only on the local operator device at compose and decrypt time.

Emergency Playbook

Recommended operator patterns when normal connectivity is unreliable or absent.

LAN available Keep one or more always-on relays on stable power. Add peer addresses manually, then let `relay-run` handle repeated syncs.
Internet partial Use the integrated relay on any reachable machine as a bridge between islands. Continue exporting bundles so that the same traffic can also move by USB if the bridge drops.
No network Export bundles, ferry them physically, and import them into a relay or store on the next reachable node. Do not copy internal store files directly.
Current desktop-shell limit This shell does not yet roam between Wi-Fi networks automatically. Aggressive network switching remains a later platform-specific transport feature.
